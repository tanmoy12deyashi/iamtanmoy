What is HTTP?

HTTP stands for Hyper Text Transfer Protcol. It is an application-layer protocol for transmitting hypermedia documents, such as HTML(Hyper Text Markup Language). It was designed for communication between web browsers and web servers, but it can also be used for other purposes. HTTP is a stateless protocol, meaning that the server does not keep any data (state) between two requests.

What Are HTTP Request Methods?

There are various types of HTTP request methods, but each one is assigned for a specific purpose. Request methods are case-sensitive, and should always be noted in upper case.

Various Types of HTTP Request Methods

  • GET
  • POST
  • PUT
  • HEAD
  • DELETE
  • PATCH
  • TRACE
  • CONNECT
  • OPTIONS

The two most common HTTP methods are: GET and POST.

GET Method

GET method is used to retrieve whatever information is identified by the Request-URL.
The query string (name & value pairs) is sent in the URL of a GET request:
/text.php?name1=value1&name2=value2

  • GET requests can be cached
  • GET requests remain in the browser history
  • GET requests can be bookmarked
  • GET requests should never be used when dealing with sensitive data
  • GET requests have length restrictions
  • GET requests are only used to request data (not modify)
Some Extra Info about GET method
  • GET Request has body: No
  • Successful response has body: Yes
  • Safe Methods: Yes
  • Idempotent: Yes
  • Cacheable: Yes
  • Allowed in HTML forms: Yes

POST Method

Another popular HTTP request method is POST. In web communication, POST requests are utilized to send data to a server to create or update a resource. The information submitted to the server with POST request method is archived in the request body of the HTTP request. The HTTP POST method is often used to send user-generated data to a server. One example is when a user uploads a profile details.

  • POST requests are never cached
  • POST requests do not remain in the browser history
  • POST requests cannot be bookmarked
  • POST requests have no restrictions on data length
Some Extra Info about POST method
  • POST Request has body: Yes
  • Successful response has body: Yes
  • Safe Methods: No
  • Idempotent: No
  • Cacheable: Only if freshness information is included
  • Allowed in HTML forms: Yes

PUT Method

PUT is used to send data to a server to create/update a resource.
The difference between POST and PUT is that PUT requests are idempotent. That is, calling the same PUT request multiple times will always produce the same result. In contrast, calling a POST request repeatedly have side effects of creating the same resource multiple times.

Some Extra Info about PUT method
  • PUT Request has body: Yes
  • Successful response has body: No
  • Safe Methods: No
  • Idempotent: Yes
  • Cacheable: No
  • Allowed in HTML forms: No

HEAD Method

HEAD is almost identical to GET, but without the response body.
In other words, if GET /users returns a list of users, then HEAD /users will make the same request but will not return the list of users.
HEAD requests are useful for checking what a GET request will return before actually making a GET request - like before downloading a large file or response body.

Some Extra Info about HEAD method
  • HEAD Request has body: No
  • Successful response has body: No
  • Safe Methods: Yes
  • Idempotent: Yes
  • Cacheable: Yes
  • Allowed in HTML forms: No

DELETE Method

Just as it sounds, the DELETE request method is used to delete resources indicated by a specific URL. Making a DELETE request will remove the targeted resource.

Some Extra Info about DELETE method
  • DELETE Request has body: May be
  • Successful response has body: May be
  • Safe Methods: No
  • Idempotent: Yes
  • Cacheable: No
  • Allowed in HTML forms: No

PATCH Method

A PATCH request is similar to POST and PUT. However, its primary purpose is to apply partial modifications to the resource. And just like a POST request, the PATCH request is also non-idempotent. Additionally, unlike POST and PUT which require a full user entity, with PATCH requests, you may only send the updated username.

Some Extra Info about PATCH method
  • PATCH Request has body: Yes
  • Successful response has body: Yes
  • Safe Methods: No
  • Idempotent: No
  • Cacheable: No
  • Allowed in HTML forms: No

TRACE Method

TRACE requests are used to invoke a remote, application loop-back test along the path to the target resource. The TRACE method allows clients to view whatever message is being received at the other end of the request chain so that they can use the information for testing or diagnostic functions.

Some Extra Info about TRACE method
  • TRACE Request has body: No
  • Successful response has body: No
  • Safe Methods: Yes
  • Idempotent: Yes
  • Cacheable: No
  • Allowed in HTML forms: No

CONNECT Method

The HTTP CONNECT method starts two-way communications with the requested resource. It can be used to open a tunnel.
For example, the CONNECT method can be used to access websites that use SSL (HTTPS). The client asks an HTTP Proxy server to tunnel the TCP connection to the desired destination. The server then proceeds to make the connection on behalf of the client. Once the connection has been established by the server, the Proxy server continues to proxy the TCP stream to and from the client.
CONNECT is a hop-by-hop method.

Some Extra Info about CONNECT method
  • CONNECT Request has body: No
  • Successful response has body: Yes
  • Safe Methods: No
  • Idempotent: No
  • Cacheable: No
  • Allowed in HTML forms: No

OPTIONS Method

The HTTP OPTIONS method requests permitted communication options for a given URL or server. A client can specify a URL with this method, or an asterisk (*) to refer to the entire server.

Some Extra Info about OPTIONS method
  • OPTIONS Request has body: No
  • Successful response has body: Yes
  • Safe Methods: Yes
  • Idempotent: Yes
  • Cacheable: No
  • Allowed in HTML forms: No
